# /user-fb - ユーザーフィードバック分析

「ユーザーが18時間使い続けたくなるアプリを作る」への達成度を、VISION.mdに基づいて評価します。

## 重要原則

**初心者100% → 中級者 → 上級者の順で改善**

複雑な機能を先に作ると初心者が混乱するため、段階的に進めます。

## 実行手順

### Phase 0: VISION.mdと前回記録の読み込み

1. `VISION.md` を読み込み、以下を取得:
   - 解決したい課題（4つ）
   - Core Value（4つの判断基準）
   - コアフロー（6ステップ）
   - 禁止事項（4つ）
2. `.claude/user-feedback/` ディレクトリを確認
3. 最新のファイルがあれば読み込み
4. 前回のスコア（初心者/中級者/上級者）を取得

### Phase 1: 並列分析（11 Subagent）

以下の11個のSubagentを**並列で起動**:

#### Subagent 1: beginner-ux（初心者向け評価）
```
subagent_type: general-purpose
model: haiku
prompt: |
  初心者向けUXを評価してください。初めてアプリを使う人の視点です。

  対象: src/app/**/page.tsx, src/components/**/*.tsx

  ## 評価軸（各25点、計100点）

  ### 1. 直感性 (25点)
  検出パターン:
  - アイコンのみのボタン（テキストラベルなし） → -3点/件
  - 専門用語・略語の使用 → -2点/件
  - 色だけで状態を表現（色弱対応なし） → -3点/件
  ベースライン: 25点から減点

  ### 2. 導線の明確さ (25点)
  検出パターン:
  - 空状態で次のアクションが不明 → -5点/件
  - ステップ数が不明な長いフォーム → -3点/件
  - 「戻る」方法が不明確 → -3点/件
  ベースライン: 25点から減点

  ### 3. エラーの優しさ (25点)
  検出パターン:
  - 「エラーが発生しました」のみのメッセージ → -5点/件
  - 技術的なエラーコード表示 → -3点/件
  - 解決方法の提示なし → -4点/件
  ベースライン: 25点から減点

  ### 4. 安心感 (25点)
  検出パターン:
  - 保存状態が不明（自動保存の表示なし） → -4点/件
  - 破壊的操作に確認ダイアログなし → -5点/件
  - 処理中の状態が分からない → -3点/件
  ベースライン: 25点から減点

  報告形式:
  ---
  ## beginner-ux 結果
  初心者スコア: [X]/100

  ### 直感性: [X]/25
  - ✅ [良い点]
  - ❌ [問題] [ファイル:行] [説明] (-X点)

  ### 導線の明確さ: [X]/25
  ...

  ### エラーの優しさ: [X]/25
  ...

  ### 安心感: [X]/25
  ...

  ### 次に取り組むべき (Top 3)
  1. [問題] - [改善方法] (+X点見込み)
  ---
```

#### Subagent 7: perceived-value（課題解決度評価）
```
subagent_type: general-purpose
model: haiku
prompt: |
  このアプリが「何の課題を解決するか」をユーザー視点で分析し、解決度を評価してください。

  対象: VISION.md, src/app/**/page.tsx, src/components/**/*.tsx

  ## このアプリの本来の目的（VISION.mdより）

  **ミッション**: 「ユーザーが18時間使い続けたくなるアプリを作る」

  **解決したい課題**:
  1. 動画制作コスト: 1本数時間→数分へ
  2. クリエイティブ検証: どれが売れるか分からない→A/Bテスト自動化
  3. PDCAが回らない: サイクルが遅い→自動データ収集
  4. 属人化: センス依存→勝ちテンプレート型化

  ## 各レベルが感じる「このアプリで解決したい課題」

  ### 初心者が感じる課題（推定）
  - 「動画を作りたいけど、どうやって作ればいいか分からない」
  - 「AI動画生成って難しそう」
  - 「商品紹介動画を簡単に作りたい」

  ### 中級者が感じる課題（推定）
  - 「もっと効率よく動画を量産したい」
  - 「前回と同じ設定で素早く作りたい」
  - 「A/Bテストで効果を比較したい」

  ### 上級者が感じる課題（推定）
  - 「大量の動画を一括管理したい」
  - 「ショートカットで爆速操作したい」
  - 「自分のワークフローをカスタマイズしたい」

  ## 評価方法

  各レベルの課題について、UIとフローから解決度を評価:
  - 100%: 課題が完全に解決されている
  - 75%: 課題はほぼ解決、一部改善余地あり
  - 50%: 課題は部分的に解決
  - 25%: 課題解決の機能はあるが使いにくい
  - 0%: 課題が解決されていない

  報告形式:
  ---
  ## perceived-value 結果

  ### 初心者が感じる課題と解決度

  | 課題 | 解決度 | 理由 |
  |------|--------|------|
  | 動画を作りたいけど方法が分からない | [X]% | [UIの該当部分と評価理由] |
  | AI動画生成が難しそう | [X]% | [UIの該当部分と評価理由] |
  | 商品紹介動画を簡単に作りたい | [X]% | [UIの該当部分と評価理由] |

  **初心者の課題解決度: [平均]%**
  最大の障壁: [具体的な問題]

  ### 中級者が感じる課題と解決度
  ...

  ### 上級者が感じる課題と解決度
  ...

  ### 総合: ユーザーが感じる価値
  - 初心者: 「[一言で表すと]」
  - 中級者: 「[一言で表すと]」
  - 上級者: 「[一言で表すと]」
  ---
```

### Phase 2: スコア計算と推奨フェーズ判定

```
初心者スコア = beginner-ux の結果 (100点満点)

VISION.md準拠度:
- 課題解決度 = vision-alignment の結果 (100点満点)
- コアフロー完走率 = core-flow-analyzer の結果 (100点満点)
- 禁止事項違反 = anti-pattern-checker の違反件数

Core Value準拠度（perceived-value + 各Subagentから算出）:
- UX中毒性 = (継続トリガー + 成功体験 + ゲーミフィケーション要素) / 3
- 操作の快感 = (クリック数 + レスポンス速度 + アニメーション) / 3
- フィードバック = (待ち時間体験 + 成功/エラー表示 + 進捗表示) / 3
- シンプルさ = (導線の明確さ + 選択肢の少なさ + 直感性) / 3

推奨フェーズ判定:
- 初心者 < 80点 → 「初心者向け改善を優先してください」

- 中級者 >= 80点 → 「上級者向け改善を開始できます」
```
